# Guides

Guides teach you **how to build applications** with bootstack — workflow-oriented,
example-driven, opinionated. Each guide covers one topic end-to-end with runnable
examples you can adapt directly.

---

## Where things live

bootstack documentation is organized so that each topic has one canonical home.

- **Guides** teach you how to build. Read these first.
- **Widgets** document what each widget does and its full parameter list.
- **Design System** catalogs the design tokens — colors, typography, variants, icons.
- **Platform** explains Tk/ttk fundamentals. Read when integrating with raw Tk or
  debugging deep layout and styling behavior.
- **[Tooling](../tooling/cli.md)** covers the CLI, project scaffolding, and packaging.
- **API Reference** is the autogenerated source of truth for every public class,
  function, and mixin. The **Capabilities** subsection documents widget-trait mixins
  (`bind`, `after`, `focus`, `grab`, `pack`, `grid`) for users who need exact mixin
  signatures.

When a topic appears in two places, the **guide is canonical** and the other pages
link to it.

## How to use guides

1. Start with the guide that matches your current task.
2. Adapt the examples — they are all runnable `bs.App(...)` snippets.
3. Follow links to Widgets, Design System, and API Reference for parameter details.
4. Return to Guides when you encounter new patterns.

If you're new to bootstack, read [App Structure](app-structure.md) first to understand
how applications are organized, then [Layout](layout.md) for the recommended approach
to building UIs.
