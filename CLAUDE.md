# bootstack — Claude Handoff

## Project overview

bootstack is a batteries-included Python/Tkinter desktop UI framework targeting
scientific, engineering, and utility applications. It provides themed widgets,
reactive state via signals, layout containers, an AppShell navigation pattern,
and a CLI (`bootstack start`, `bootstack gallery`, etc.).

- **Active branch:** `docs/restructure` — single long-lived branch for the docs
  initiative. Stay on it. PRs go `docs/restructure` → `main`, never per-session
  branches to `main`.
- **Docs build tool:** `zensical` (config `zensical.toml`). Build: `zensical build`.
  Preview: `zensical serve`.
- **Link validation is disabled** (`invalid_links = false` in `zensical.toml`).
  A clean build does NOT prove links are valid. After any deletion, grep for
  the deleted paths and repoint inbound links manually.

## Docs restructure — settled decisions

Do not relitigate these.

1. **`docs/reference/capabilities/` stays.** Autogenerated mixin reference — the
   legitimate use of "capability".
2. **`docs/capabilities/` (narrative essay tree) is fully dissolved.** Done.
3. **Guides are canonical.** When a topic appears in two places, the guide wins.
   All cross-references from `widgets/`, `design-system/`, `platform/` point to guides.
4. **Code samples use `bs.*`, never `ttk.*`.** Fix on sight.
5. **`docs/tooling/` pages stay.** Link to them from guides; no parallel `distribution.md`.
6. **No redirect plumbing needed.** Docs aren't public yet. Grep-and-repoint is sufficient
   for internal link integrity.
7. **Icon docs are self-sufficient.** Name Bootstrap Icons as the source, link
   https://icons.getbootstrap.com/, mention `ttkbootstrap_icons_bs` as an
   implementation detail only, document `bootstack gallery` as the in-project explorer.
8. **Widget pages use a flat catalog structure with tabs.** No umbrella `## Appearance`
   / `## Examples & patterns` / `## Behavior` headings — their former subsections
   become flat h2s. Each page has a `=== "Documentation"` tab (cookbook content)
   and a `=== "API"` tab (autogenerated mkdocstrings). Template:
   `docs_templates/widget-page-template.md`. Tabs config lives in `zensical.toml`
   (`pymdownx.tabbed` extension + `content.tabs.link` theme feature).
9. **The widget-page restructure is single-phase, gated on the mkdocstrings fix.**
   The `:::` directive currently renders only `__init__` because
   `inherited_members = false` hides ttk methods. Until that's resolved, no widget
   pages get migrated to the tabbed template. See memory
   `project_mkdocstrings_api_incomplete.md`.
10. **"Consider a different control when..." is opt-in by widget family,
    not on every page.** Include only where readers can plausibly land on the
    wrong widget in a family of overlapping controls (TreeView/TableView/ListView,
    SelectBox/OptionMenu/Combobox, etc.). For simple widgets (Button, Label,
    Separator), omit. Format: condition-first lowercase, em-dash separator, no
    bold prefix — see template.
11. **"Related widgets" footer is dropped.** Nav sidebar provides browse-style
    discovery. Bottom of page is a lean `## See also` with guide links only;
    API reference lives in the API tab.
12. **Badge is text-only by design.** TBadge ttk style layout omits the image
    element, so `icon=`, `icon_only=`, `image=`, `compound=` are silently dropped
    even though Label accepts them. Do not document Badge as supporting icons.
13. **Image reuse during page migration is selective, not automatic.** When
    migrating a widget page to the new template, only the quickstart screenshot
    is auto-carried. All other existing screenshots (variants, formats, addons,
    states, etc.) are evaluated case-by-case during the page rewrite — drop the
    ones that no longer fit the new section structure or that the new content
    doesn't need. Do not mechanically transplant every `<img>` from the old page.

## Key API notes

```python
import bootstack as bs

# App
app = bs.App(title="...", size=(w, h))
app.mainloop()

# AppShell
shell = bs.AppShell(title="...", size=(w, h))
page = shell.add_page("key", text="Label", icon="icon-name")
shell.toolbar.add_button(icon="...", command=...)

# Layout
bs.PackFrame(parent, direction="vertical|horizontal|row|column|row-reverse|column-reverse",
             gap=N, padding=N, fill_items="x|y|both|none",
             expand_items=True, anchor_items="n|ne|e|se|s|sw|w|nw|center")
bs.GridFrame(parent, rows=N|[...], columns=N|[...], gap=N|(col_gap, row_gap),
             sticky_items="ew|nsew|...", auto_flow="row|column|row-dense|column-dense|none")
# Size specs in rows/columns: int weight, "auto", "100px"
bs.Card(parent)  # accent='card', show_border=True, padding=16

# Signals
sig = bs.Signal(value)  # widgets accept signal=sig or textsignal=sig

# Icons — Bootstrap Icons catalog, any name works
bs.Button(parent, icon="house")   # https://icons.getbootstrap.com/
# In-project explorer: bootstack gallery → Icons page

# Color tokens
accent="primary|secondary|success|warning|danger|info|light|dark"
variant="solid|outline|ghost|link|toggle"
surface="content|card|chrome|overlay|input"
# Modifiers: "primary[+1]", "primary[500]", "primary[subtle]", "primary[100][muted]"

# Font tokens
font="body"  |  font="heading-lg[bold]"  |  font="body+2[italic]"
```

### API gotchas (surfaced during session work)

- **`bs.App` does not accept `theme=`, `locale=`, `light_theme=`, `dark_theme=`
  as direct kwargs.** These go through `settings={"theme": "..."}` or an
  `AppSettings` instance. Exception: `localize=` IS a direct kwarg.
- **`bs.Form` uses `col_count=`, not `columns=`.**
- **`Form.validate()` only fires rules with trigger `always` or `manual`.** Rules
  triggered by `blur` or `key` are skipped.
- **`Dialog.show()` / `FormDialog.show()` return `None`** and set `self.result`.
  Use `dlg.show(); if dlg.result: ...` — not `result = dlg.show()`.
- **`bs.TableView` only accepts `SqliteDataSource`**, not `MemoryDataSource` or
  `FileDataSource`. For other sources, format records and load them directly.
- **`bs.TableView` has no per-column formatter.** Format records before loading.
  For per-row color, drop to `bs.TreeView`.
- **`bs.SelectBox` emits `<<Change>>`**, not `<<SelectionChange>>`.
- **All Field-based widgets emit `<<Change>>`** (not `<<Changed>>`) and `<<Validate>>`
  (not `<<Validated>>`). Using the wrong name silently produces no callbacks.
- **`on_changed`/`on_input` callbacks receive a Tkinter event object** (`event.data["value"]`).
  **`on_valid`/`on_invalid`/`on_validated` callbacks receive a plain dict** (`data["value"]`).
  These two groups have different shapes — do not use the same callback signature for both.
- **`text=Signal(...)` does NOT work for reactive labels.** Use `textsignal=signal` instead.
  Passing a Signal as `text=` displays the Signal's repr as static text.
- **`bs.TreeView` not `bs.Treeview`.** Capital V — `bs.Treeview` does not exist.
- **`Meter` deprecated params:** `amountused`, `amounttotal`, `subtext`, `stripethickness`
  still work but trigger `DeprecationWarning`. Use `value`, `maxvalue`, `subtitle`,
  `segment_width` instead.
- **`value=` is ignored when `signal=` or `variable=` is also passed** on CheckButton,
  CheckToggle, Switch, RadioButton, RadioToggle. To seed a signal's initial value,
  set it on the Signal itself: `bs.Signal(True)` not `bs.Signal(False)` + `value=True`.
- **`ToggleGroup(padding=N)` raises `TypeError`** — source bug, do not pass `padding=`.
- **`insert_addon` with `bs.CheckButton` raises `TclError`** — `density=` leaks to ttk.
  Use `bs.Button` for addon toggles instead.
- **Localization domain mismatch:** `bootstack add i18n` scaffolds domain
  `"messages"`, but `bs.App` auto-inits domain `"bootstack"`. User-app translations
  are silently inactive until the app re-calls
  `MessageCatalog.init(locales_dir="locales", domain="messages", default_locale="...")`.
  Doing so deactivates the framework catalog (only one gettext domain active at a time).
- **`bs.L(key, *fmtargs)` uses positional `%s`/`%d` format args**, not `{name}`
  kwargs. Returns a `LocalizedTextSpec`, not a string.
- **`bs.LV(value, format_spec)` is for value formatting** (numbers, dates,
  currency) — not text translation.
- **`bs.Form` button command lambda receives the form as first arg:**
  `lambda f: print(f.value)`, not `lambda: print(form.value)`.
- **`RadioGroup.set()` validates against keys, not values.** When `key=` differs
  from `value=` in `add()`, pass the key to `set()`, not the value.
