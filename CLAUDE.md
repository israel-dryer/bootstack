# bootstack — Claude Handoff

## Project overview

bootstack is a batteries-included Python/Tkinter desktop UI framework targeting
scientific, engineering, and utility applications. It provides themed widgets,
reactive state via signals, layout containers, an AppShell navigation pattern,
and a CLI (`bootstack start`, `bootstack gallery`, etc.).

- **Working directory:** `D:\Development\bootstack`
- **Active branch:** `docs/restructure` — single long-lived branch accumulating
  all sessions in this initiative. Stay on it; do not branch per session. Open
  one PR at the end (or per-session PRs back to `docs/restructure`, never to
  `main`, until the initiative is complete).
- **Docs build tool:** `zensical` (config is `zensical.toml`, not `mkdocs.yml`).
  Build locally with `zensical build` or preview with `zensical serve`.
- **Heads-up:** `zensical.toml` has `[project.validation] invalid_links = false`,
  so a clean build does **not** prove links are valid. After deletion passes,
  manually grep for the deleted paths and verify nothing inbound is orphaned.

## Current initiative: Docs restructure

The guides/ tree has structural problems. A user looking up a single topic
(spacing, color, signals, icons) currently lands on 2–4 pages across `guides/`,
`capabilities/`, `platform/`, and `design-system/`. We're consolidating.

Full architectural review is in the conversation that produced this plan; the
material decisions are captured below — **do not relitigate them.**

### Settled architectural decisions

1. **`docs/reference/capabilities/` stays as-is.** That's the autogenerated
   mixin reference (`::: bootstack.core.capabilities.PackMixin` etc.) and is
   the legitimate use of "capability" — widget-trait reference.
2. **`docs/capabilities/` (the narrative essay tree) is dissolved.** Most pages
   duplicate guides or are topical essays. Some get folded into guides; most
   get deleted; one (validation) gets promoted to a guide.
3. **Guides are canonical.** When a topic appears in both a guide and an essay,
   the guide wins. Cross-references from `widgets/`, `design-system/`,
   `platform/` etc. all point to guides.
4. **`styling.md` + `theming.md` merge into `color-and-theming.md`.** The
   current split is artificial and forces readers to bounce.
5. **`spacing-and-alignment.md` is rewritten as a bootstack-vocabulary-first
   practical guide** ("Path A"). The current Tk-mechanics-first framing moves
   to a demoted "Under the hood" section.
6. **Code samples use `bs.*`, never `ttk.*`.** Several guides still have stale
   `ttk.Button` etc. — fix on sight.
7. **Existing `docs/tooling/` pages stay.** They cover CLI, project structure,
   and build/ship adequately; we link to them from guides rather than create
   a duplicate `distribution.md`.
8. **Docs are not publicly released yet.** No external bookmarks, no SEO, no
   redirect plumbing required. Delete pages freely. The only link integrity
   that matters is **internal** — i.e., other doc pages that still reference
   a deleted path. Grep-and-repoint is sufficient; no `mkdocs-redirects` or
   equivalent is needed.
9. **Icon docs must be self-sufficient.** bootstack uses Bootstrap Icons via
   the `ttkbootstrap_icons_bs` provider (see `src/bootstack/style/bootstyle_builder_base.py`).
   Readers should not have to navigate to the ttkbootstrap project, the
   `ttkbootstrap_icons_bs` package, or anywhere else to learn how to use
   icons in bootstack. The icon docs must:
   - Name the underlying source: **Bootstrap Icons**.
   - Link to the Bootstrap Icons website — https://icons.getbootstrap.com/ —
     as the canonical "what icons can I use?" reference.
   - Document the in-project icon explorer: the Icons page of the widget
     gallery, launched via `bootstack gallery`. Currently lives in
     `src/bootstack/cli/demo.py::_build_icons_page` and showcases ~20 common
     icons, sizes, and accent-colored variants.
   - Note that any icon name from the Bootstrap Icons catalog works as the
     `icon=` value (e.g. `icon="house"`, `icon="exclamation-triangle"`).

## Reader-facing mental model (use this wording in `guides/index.md`)

> **Guides** teach you how to build. Read these first.
> **Widgets** document what each widget does and its parameters.
> **Design System** catalogs the design tokens (colors, typography, variants).
> **Capabilities** (under Reference) document widget-trait mixins (`bind`,
> `after`, `focus`, `grab`, `pack`, `grid`) for users who need precise mixin
> behavior.
> **Platform** explains Tk/ttk fundamentals — read when integrating with raw
> Tk or debugging deep behavior.
> **Tooling** covers the CLI and packaging.
> **API Reference** is the autogenerated source of truth.
>
> When a topic appears in two places, the **guide is canonical** and the
> others link to it.

## Key API notes (standing reference)

- Import: `import bootstack as bs`
- App entry: `app = bs.App(title="...", size=(w, h))` then `app.mainloop()`
- AppShell: `shell = bs.AppShell(title="...", size=(w, h))`,
  `page = shell.add_page("key", text="Label", icon="icon-name")`,
  `shell.toolbar.add_button(icon="...", command=...)`
- PackFrame: `bs.PackFrame(parent, direction="vertical|horizontal|row|column|row-reverse|column-reverse",
  gap=N, padding=N, fill_items="x|y|both|none", expand_items=True,
  anchor_items="n|ne|e|se|s|sw|w|nw|center")`
- GridFrame: `bs.GridFrame(parent, rows=N|[...], columns=N|[...], gap=N|(col, row),
  sticky_items="ew|nsew|...", auto_flow="row|column|row-dense|column-dense|none")`
  - Size specs in `rows`/`columns`: int weight, `"auto"`, `"100px"`
- Card: `bs.Card(parent)` — Frame with `accent='card'`, `show_border=True`,
  `padding=16` defaults
- Signals: `sig = bs.Signal(value)` — widgets accept `signal=sig`
- Icons: Bootstrap Icons by name (e.g. `"house"`, `"table"`, `"bar-chart"`,
  `"gear"`, `"sun"`, `"bell"`)
  - Catalog: https://icons.getbootstrap.com/ — any name there works
  - In-project explorer: `bootstack gallery` (Icons page)
  - Underlying provider: `ttkbootstrap_icons_bs` (do not require users to
    learn this package; it's an implementation detail)
- Color tokens: `accent="primary|secondary|success|warning|danger|info|light|dark"`,
  `variant="solid|outline|ghost|link|toggle"`,
  `surface="content|content[1]|chrome|overlay|titlebar"`
- Color modifiers (chained brackets): `"primary[+1]"`, `"primary[500]"`,
  `"primary[subtle]"`, `"primary[100][muted]"`
- Font tokens: `font="body"`, `font="heading-lg[bold]"`, `font="body+2[italic]"`
